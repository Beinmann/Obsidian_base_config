# Obsidian Settings

My Obsidian vault config — settings, templates, and snippet/plugin choices that make it feel like home. Public so I can reference it from new machines, and so anyone curious can borrow bits.

This repo is **not** a working copy of the vault. The vault itself isn't a git repo (notes stay out of version control). Only the config files, templates, and a handful of empty placeholder pages live here.

## Vault layout

Top-level `.md` files are capture targets used by [QuickAdd](#plugins). Folders are storage.

| Path | Role |
|------|------|
| `Braindump.md` | Quick-capture inbox. New thoughts append under `# New` with a timestamp. |
| `RAM.md` | Daily checklist. Items append under `### DD.MM.YYYY`. |
| `done, notes to self and more list or log.md` | Done log + self-notes (A/B sides) + nice-things-to-do entries. |
| `Main.md` | Index linking Braindump / RAM / log / Daily Notes MOC. |
| `Daily Notes/` | Daily notes folder (tagged `#dailyNote`). |
| `Daily Notes MOC.md` | Dataview query listing every daily note. |
| `Permanent_notes/` | Zettelkasten-style notes named `PN - YYMMDDHHmmss`. |
| `Main/` | Catch-all for misc notes that don't fit elsewhere. |
| `Templates/` | Templater + QuickAdd templates. |
| `init.vim` | Vim config applied by the Vimrc Support plugin. |

## Plugins

Community plugins enabled in this config:

- **QuickAdd** — capture macros for Braindump / RAM / log; the *Helper - Create Permanent Note* macro generates a timestamped PN file in `Permanent_notes/`.
- **Templater** — template engine for everything in `Templates/`.
- **Dataview** — powers the Daily Notes MOC and any inline queries.
- **Excalidraw** — drawings (stored under `Plugins/Excalidraw/` inside the vault).
- **Obsidian Git** — vault backups (`vault backup: {{date}}`, auto-pull on boot).
- **Vimrc Support** — applies `init.vim`.
- **Scroll Offset** — keeps the cursor off the screen edge while scrolling.
- **Calendar** — date picker for daily notes.

## Snippets

`.obsidian/snippets/main.css` and `snippet.css` are mine and ship with this repo.

`Dune-mermaid.css` is **not** committed — it's the companion snippet for the Dune theme by Jopp ([GPL-2.0](https://github.com/Jopp-gh/Obsidian-Dune84)). To install:

1. Grab `Dune-mermaid.css` from [Jopp-gh/Obsidian-Dune84/snippets/](https://github.com/Jopp-gh/Obsidian-Dune84/tree/main/snippets).
2. Drop it in `.obsidian/snippets/` and toggle it on in *Settings → Appearance → CSS snippets*.
3. Optional — apply the two patches below if you want my edge-label / dark-mode arrow tweaks.

### Local patches to Dune-mermaid.css

Two small legibility tweaks. Both target dark-mode arrow / edge-label contrast.

**Edge label color** (around line 672):

```css
/* upstream */
.mermaid .edgeLabel {
  color: var(--text-normal) !important;
}

/* patched */
.mermaid .edgeLabel,
.mermaid .edgeLabel tspan {
  color: #111 !important;
}
```

**Dark-theme flowchart arrows** (around line 722):

```css
/* upstream */
.theme-dark .mermaid path:is(.flowchart-link, .transition) {
  stroke: var(--text-normal) !important;
  filter: contrast(0.2) brightness(0.2) !important;
}

/* patched */
.theme-dark .mermaid path:is(.flowchart-link, .transition) {
  stroke: #cccccc !important;
  filter: none !important;
}
```

The second patch undoes upstream's heavy darkening that made arrows nearly invisible in dark mode.

## Syncing across machines

> **WIP / idea — not implemented.** Today the workflow is fully manual.

Today's manual flow:

1. Clone this repo into a scratch directory **next to** (not inside) the target vault.
2. Copy the vault's config files over the cloned repo (or vice versa, depending on direction).
3. `git diff` in the scratch dir to review.
4. Manually pick which changes to keep, commit, push.
5. On a fresh machine, copy the relevant files from the cloned repo into the new vault's `.obsidian/`, `Templates/`, etc.

**Planned**: a small `sync.sh push <vault>` / `sync.sh pull <vault>` script that rsyncs a curated allowlist of paths (`.obsidian/*.json`, `.obsidian/plugins/*/data.json`, `.obsidian/snippets/main.css`, `.obsidian/snippets/snippet.css`, `Templates/`, `init.vim`, `.gitignore`) between vault and repo. Would collapse steps 1–2 and 5 into one command. Not written yet.
